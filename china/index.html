工作方法
我的工作方法。
真正决定方案质量的，往往不是先选模型，而是先把几个前提判断清楚：谁在使用、业务价值是否明确、失败成本是否可控，以及输出能否进入真实工作流。只有这些问题被定义清楚，AI 方案设计才具备产品意义。

从问题定义到方案设计

  • 先定义业务目标、目标用户、成功指标
  • 判断场景是否真的适合 AI
  • 设计用户旅程、数据与上下文体系
  • 决定模型方案、guardrail、fallback 和 human-in-the-loop

从上线治理到结果闭环

  • 建立 golden set、shadow audit、线上监控
  • 同时看质量、时延、成本、风险、业务指标
  • 通过灰度上线、bad case、A/B test 持续迭代
  • 把模型输出变成可被业务稳定消费的产品能力
旗舰案例
Hybrid LLM Brand Classification
这个案例要证明的，不是“LLM 可以做分类”，而是 AI 产品经理如何把模型能力重构为可上线的系统能力：用结构化上下文提升判断质量，用受限输出保证可消费性，用缓存 serving 与 fallback 机制保护实时链路，再用评测与监控体系把模型输出纳入真实业务治理。
业务问题

query-brand 关系判断直接影响广告相关性、CTR、排序质量与广告收入。问题不只是提升语义理解，而是如何在高并发拍卖链路里引入更强判断能力，同时不突破时延、成本与可用性边界。

我的核心贡献

核心贡献不在于参与一个 LLM 项目，而在于将 LLM 推理能力从单点模型能力转化为可进入真实业务流程的产品能力。关键工作包括定义受限输出结构、设计结构化上下文组装方式、建立 Redis-backed serving 与 fallback 机制，并将评测、审计与监控前置为上线标准。

方案判断

采用 `offline / near-offline LLM + Redis + legacy fallback` 的 hybrid 方案，核心判断不是技术偏好，而是产品取舍：把昂贵推理与实时 serving 解耦，使高价值决策能够毫秒级命中，同时在 cache miss 时保留确定性 fallback，保证拍卖链路稳定可用。

评测与上线

上线策略依赖的不是单次模型表现，而是完整治理机制：50,000 query golden dataset、schema validation、hallucination check、1% shadow audit，以及 latency、fallback、cache hit、invalid output rate、label distribution 等线上监控，确保输出可评测、可审计、可回退。

  • 约 $130M revenue impact

我的核心贡献

这一问题并未被视为单一的 churn model 项目，而是被定义为生命周期预警产品。关键判断在于：漏掉高风险客户的业务代价高于误报，因此指标设计必须以 Recall-first 为核心；后续交付的重点也不是模型本身，而是让风险信号真正进入运营与客户成功团队的工作流。

产品化标准
我如何判断 AI 是否真的可以上线。
在大多数团队里，真正拉开差距的不是“有没有接入模型”，而是有没有把上线门槛、消费方式和回退机制定义清楚。下面这套标准，是我判断 AI 产品能否进入真实业务流程的基本框架。

输出必须可被消费

  • 输出必须有明确边界，适合下游系统或业务角色直接消费
  • 低置信度、异常结果与不可解释输出必须有 fallback 或人工接管路径
  • AI 能力的价值不在单次回答，而在能否稳定进入真实 workflow
  • 模型输出要服务于排序、筛选、预警、推荐或动作触发，而不是停留在展示层

上线必须可被治理

  • 上线前要有明确的 golden set、评测标准和 bad case taxonomy
  • 上线中要能监控质量、时延、成本、fallback rate 与异常分布
  • 上线后要能通过 shadow audit、抽样复核和版本管理持续迭代
  • 真正可上线的 AI 产品，必须是可评测、可审计、可回退、可复用的系统能力
落地能力
需求到上线的推进方式。
这套方法覆盖了从需求定义、方案设计、评测验收，到灰度发布、交付协同与风险治理的完整落地闭环。

PRD / 原型 / 验收思路

  • 需求定义：明确目标用户、核心场景、能力边界、人工介入点与非目标范围
  • 方案表达：能把模糊业务需求拆成产品流程、交互逻辑、模型链路和上下文来源
  • 验收标准：不仅写功能是否完成，也写 accuracy、latency、fallback rate、人工接管率等 AI 专项指标
  • 版本规划：优先用 MVP 验证核心价值，再决定是否扩到 Agent、多工具调用或更复杂的自动化链路

风险控制 / 灰度 / 交付思路

  • 风险控制：幻觉防控、输出边界、权限控制、人工复核点、异常回滚机制
  • 评测上线：golden set、shadow audit、bad case 分类、线上监控和灰度发布
  • 跨团队协同：能和算法、工程、测试、运营、销售或实施团队对齐目标与优先级
  • 交付闭环：把客户反馈、标注修正、模型优化和产品迭代串起来，形成持续进化机制
作品集集中呈现的，是一类在 AI 产品岗位中最具区分度的能力：将概念验证转化为可上线、可协同、可复用并能持续产生业务结果的产品能力。
